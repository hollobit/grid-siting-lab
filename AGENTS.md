# 에이전트 활용 및 작업 규칙

## 현재 런타임

- 이 세션에는 별도 서브에이전트 생성/위임 도구가 노출되어 있지 않다.
- 따라서 메인 에이전트가 구현을 담당하고, 독립적인 자료 조회·파일 점검·검증 명령은 가능한 경우 병렬 도구 호출로 처리한다.
- 사용자가 요청한 “멀티 에이전트” 의도는 역할 분리와 문서화로 보존한다.

## 역할 분리

### Main / Product

- 사용자 요구사항을 제품 구조와 UI 흐름으로 번역한다.
- 점수 모델과 시뮬레이션의 의미를 결정하고, 확정 데이터와 프록시 데이터를 구분한다.

### Data / Research

- OpenInfraMap·OSM·공공 데이터 출처와 날짜를 확인한다.
- 도시, 교통, 철도, 공항, 통신, 위성, 클라우드/IDC, 대학/R&D 레이어의 데이터 연결 지점을 기록한다.

### Frontend / Visualization

- 지도·레이어·후보지 카드·시나리오 패널을 구현한다.
- Leaflet 외부 리소스가 실패해도 fallback map이 동작하도록 한다.

### QA / Handoff

- 정적 서버 실행, 콘솔 오류, 반응형 레이아웃, 후보지 선택, 슬라이더, 레이어 토글을 검증한다.
- 결과·한계·다음 데이터 연동 지점을 `HISTORY.md`에 남긴다.

## 병렬 작업 규칙

- 읽기 전용 검색/점검은 병렬 처리한다.
- 같은 파일을 여러 작업이 동시에 수정하지 않는다.
- 파일 변경은 `apply_patch`로만 수행한다.
- 각 의미 있는 변경은 `HISTORY.md`에 짧게 기록한다.
- 새 외부 데이터는 화면 문구·README·출처 링크에 반영한다.
