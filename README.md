# GRID / SITING LAB

대한민국 전력망을 출발점으로 AIDC(Artificial Intelligence Data Center) 후보지의 초기 적합도를 비교하고, IT load·PUE·이중화·부하율·냉각 방식 조건에 따른 전력·용수·탄소·비용을 시뮬레이션하는 정적 웹 프로토타입입니다.

## 실행

```bash
npm run dev
```

`http://localhost:4173`에서 확인할 수 있습니다. 별도 빌드 도구 없이 동작하며, 지도 배경이 연결되지 않을 경우에도 내장된 전력망 개략도가 표시됩니다.

GitHub Pages: https://hollobit.github.io/grid-siting-lab/

## 구현 범위

- OpenInfraMap / OpenStreetMap 기준 한국 통계 스냅샷 표시
  - 발전소 4,089개 / 매핑 발전량 121,404 MW (OIM stats 2026-06-25)
  - 전력선 15,176 km
  - 154 kV 9,420 km, 345 kV 4,584 km, 765 kV 557 km
- **OSM 전국 전력망·교통 정적 스냅샷 (2026-07-15 추출)**
  - `data/kr_power_lines.json` — 송전선 3,652개 way (765 kV 41 · 345 kV 847 · 154 kV 2,102 · 기타 662)
  - `data/kr_power_plants.json` — 발전소 4,129개 (태그된 출력 합계 약 124.3 GW), 발전원별 색상·용량 비례 표시
  - `data/kr_transport.json` — 간선 철도(`railway=rail` usage=main/branch, 고속철 구분)와 고속도로(`highway=motorway`) 실측 선형. TRANSPORT(도로)·RAIL/AIR(철도) 컨텍스트 레이어에 표시
  - 후보지 반경 50 km의 매핑 발전용량·발전소 수·345 kV+ 최근접 거리를 계산해 시뮬레이션과 연동. 임의 지점 채점의 도로·철도 접근성도 실측 선형 기준
- Leaflet + OpenStreetMap 배경 지도 및 전압급별 전력망·발전원·후보지·시설·수자원 레이어
- 당진·나주·세종·이천·울산 후보지의 전력망, 수전 여유, 수자원, 통신, 재해·기후, 토지·인허가 요인 비교
- 시뮬레이션: IT load / PUE / N+0·N+1·N+2 / 평균 부하율 / 냉각 방식(WUE)
  - 연간 전력량(GWh), 필요 수전용량, 후보지 headroom
  - 연간 용수 사용량(m³ · WUE 프록시), 연간 탄소 배출(tCO₂ · 지역 탄소집약도 프록시), 연간 전력비용(억원 · 160원/kWh 프록시)
  - **CAPEX 개산(억원)**: 구축비 120억/MW 프록시 × 이중화·냉각·PUE 계수 + 계통 인입비(345 kV+ 최근접 거리 × 25억/km + 변전 2억/MW). 표준구성(N+1·하이브리드·PUE 1.25·인입 5 km) 대비 증감률 배지 표시
  - **최대 가능 IT capacity 추정(MW)**: 계통 수전 여유 프록시와 냉각 용수 가용량 프록시(LOW 300만·MED 120만 m³/년) 중 병목 기준. 현재 IT LOAD가 한도를 초과하면 경고
  - OSM 스냅샷 기반 "수요 / 주변 50 km 발전용량" 비율
- **지도 임의 지점 분석**: 후보지에서 25 km 이상 떨어진 도시·임의 위치(발전소·시설 마커 포함)를 클릭하면 반경 50 km OSM 실측(발전용량·최근접 345 kV+ 회선)과 정적 레이어 거리 프록시(도시·백본·IDC·대학·취수원)로 즉석 채점·시뮬레이션. 재해·토지 요인은 미산정 중립값으로 고지
- **SIMULATION BRIEF (자동 해석)**: 시뮬레이션 결과에서 규칙 기반으로 장점 / 단점·리스크 / 기대 컴퓨트 용량 / 개선 방향 / 지역사회 영향 설명문을 생성. 컴퓨트 용량은 H100급 가속기 환산(1.2 kW/장 · 가속기 비중 85% · FP8 2 PFLOPS/장)으로 가속기 수·EFLOPS·프론티어 클러스터 대비 규모를 제시하고, 입지 최대 추정치 기준 확장 여력도 함께 표시. 입력·후보지 변경 시 자동 갱신
- **지역사회 영향 (최대 용량 가동 가정)**: 최대 추정 용량 기준으로 주민 체감 단위 환산 — 전력수급(가구분 전력·반경 50 km 발전량 점유율·계통 경합 경고), 물공급(주민 생활용수 인구 환산·지역 가용 대비 %·가뭄기 경합 경고), 환경(tCO₂·승용차 환산, 냉각 방식별 백연/온배수/소음/열섬 쟁점, 비상발전기·송전선로 수용성), 완화 방안(폐열 재이용·중수 냉각·상생요금·고용 프록시). 환산 계수: 가구 3.6 MWh/년 · 1인 생활용수 67 m³/년 · 승용차 2 tCO₂/년
- 시설 레지스트리: lifecycle(운영/건설/계획) × 사업자 유형(hyperscaler/colocation/enterprise) 표준화, 시설별 출처·확인일·confidence 표기, 지도 FACILITIES 레이어와 필터 동기화
- 전력 / 생태계 / 회복력 / **정치적 고려** / **지역균형발전** 5개 가중치를 조절하는 적합도 모델 (정치·균형 요인은 정책 정합성·비수도권 가산 프록시)
- **SCALE MODEL FIT**: 초대형(300 MW)·대형(100 MW)·중형(40 MW)·소형(10 MW) AIDC 모델별 적합도 — 기본 적합도 × 공급충족률(계통/용수 병목) − 민원 페널티(토지 갈등·용수 경합·주민 전력 경합·대도시 인접, 규모 가중). 전력·용수 충족 + 민원 최소 전제를 만족하는 모델별 추천 입지 표시
- **규모별 전국 추천 TOP 5 (지도)**: 345 kV+ 회선 중간점 133개 시드를 임의 지점 엔진으로 채점해 클래스별 상위 5곳(전제 충족 우선 · 30 km 분산)을 SUITABILITY 레이어에 점선 링으로 표시 — 초대형 보라 · 대형 핑크 · 중형 시안 · 소형 초록. 클릭 시 해당 지점 즉석 분석, 가중치 변경 시 재순위
- **초대형 신설 후보 (발전원 직접연계형)**: 고리·서인천·한울(울진)·보령·당진 등 GW급 발전 클러스터(반경 15 km 합산 ≥ 2.5 GW) 인접지를 전용선로·직접 PPA(용량의 10% 프록시) 전제로 스크리닝 — 최대 IT 환산, 취약 요인별 개선 과제, 개선 목표치(72) 달성 시 잠재 점수(현재 → 개선 시)를 제시. 지도에 주황 대형 링으로 표시
- **GW STRATEGY**: 1.5 GW+ 허브 9곳에 정부 목표(2029 8.4 GW → 2035 18.4 GW)를 배분 — 2029(직접연계 10%)·2035(보강+신규 전원 20%) 허브별 할당, 목표 대비 부족분과 충당 수단(SMR·해상풍력·LNG 직결, HVDC, RE100 캠퍼스), 1 GW+ 최적 추천(해결 과제 포함), 단계별 달성 경로 제안
- **후보지 비교 리포트 PDF**: 후보지 × 규모 모델 매트릭스·실측·공급 한도·추천 입지를 인쇄 뷰로 생성해 브라우저 "PDF로 저장"으로 내보내기 (SHORTLIST 섹션 버튼)
- **재해 레이어 (HAZARD)**: 홍수 저지대 5곳·지진 다발권 3곳 프록시존. 임의 지점의 회복력 점수를 재해존 근접도로 산정. 홍수위험지도·기상청 공공데이터 연동 전 탐색용
- **한전 접속여유 연동 구조**: `data/kepco_capacity.json` 배치 시 변전소 여유용량으로 수전 여유를 자동 교체 (스키마: `data/kepco_capacity.sample.json`). 미연동 시 스크리닝 프록시로 동작
- 변경 이력은 `changelog.md`에 기록
- 후보지 기준 반경 50 km의 OpenStreetMap Overpass API 동기화 버튼
- 외부 지도 리소스가 불가능한 환경을 위한 내장 fallback map (스냅샷 로드 실패 시 개략도 프록시로 degrade)

## 데이터와 주의사항

화면의 국가 통계는 [OpenInfraMap South Korea Stats](https://openinframap.org/stats/area/South%20Korea)의 2026-06-25 스냅샷을 반영했습니다. OpenInfraMap은 OpenStreetMap 데이터를 직접 사용하며, 공개 매핑 데이터는 실제 네트워크의 완전한 표현이 아닐 수 있습니다.

`data/` 디렉토리의 전력망 스냅샷은 Overpass API로 2026-07-15에 추출한 OpenStreetMap 데이터(ODbL 1.0)입니다. 표시용으로 좌표를 단순화했고, 전압급은 태그된 최대 전압 기준이며, 발전소 출력은 `plant:output:electricity` 태그가 있는 경우만 합산됩니다. 시설 레지스트리의 하이퍼스케일러 리전 등 비공개 위치는 공개 자료 기반 프록시 좌표(CONF LOW)로 표시합니다. 탄소집약도·전기요금·WUE는 공개 통계 기반 탐색용 프록시입니다.

후보지 점수와 전력망 선형·발전원 위치는 UX 프로토타입을 위한 탐색용 모델입니다. 실제 사업 검토에는 한전 계통접속 가능용량·접속비, 토지/용도지역, 환경·재해, 용수 인허가, 통신사 이중화·실측, 주민·인허가 검토가 추가로 필요합니다.

## 공공 API 연동 제안

현재 프로토타입에서 바로 쓸 수 있는 API는 인증키 없이 접근 가능한 OpenStreetMap Overpass API입니다. 후보지 반경 50 km에서 `power=substation/plant`, `railway=station`, `aeroway=aerodrome`, `amenity=university`, `man_made/building=data_center` 태그를 조회해 주변 인프라 개수를 표시합니다. 호출량이 크거나 API가 제한될 때는 스냅샷으로 자동 유지합니다.

실서비스로 확장할 때는 다음 순서가 적합합니다.

1. **전력**: 한전 계통연계·변전소별 실제 여유용량/접속비를 공식 데이터 또는 사업자 협의 데이터로 보강하고, OpenInfraMap은 형상·전압급·발전원 위치의 기준 레이어로 유지합니다.
2. **지리·교통**: 국토교통부 국토교통 데이터 오픈 API, 국가교통정보센터/공공데이터의 도로·철도·공항·물류 데이터를 OSM 형상과 교차검증합니다.
3. **통신**: 과학기술정보통신부·NIA·KISA의 초고속 인터넷/국가정보통신망 공개 통계와 통신사 임대회선·해저케이블·IX 접속 정보를 별도 보안 레이어로 연결합니다.
4. **클라우드·IDC**: KISA/공공기관의 데이터센터 현황과 사업자 공개 리전·IDC·IX 정보를 시설 단위로 관리하되, 보안상 정확한 시설 위치/회선 용량은 권한 데이터로 분리합니다.
5. **위성**: 중앙전파관리소의 위성관련 무선국 공개 현황을 지상국/백업 통신 프록시로 연결하고, 가시성·기상·전파 간섭은 별도 검증합니다.
6. **환경·재해·수자원**: 기상청, 국가공간정보, 환경부·국가수자원관리 데이터의 홍수·폭염·가뭄·취수·용도지역 레이어를 추가해 적합도와 운영 리스크를 분리합니다.

API 키·쿼터·개인정보·시설 보안 이슈가 있는 데이터는 브라우저에서 직접 호출하지 않고 서버 캐시/ETL로 주기적으로 정규화하는 구성이 안전합니다.

## 기존 AI Data Center Map과의 연계 검토

`aidatacentermap.org/map`은 검색 색인상 미국 데이터센터를 운영/계획/하이퍼스케일러 상태와 총 전력량으로 탐색하고, 수자원·대수층·전력망 영향과 지역 영향 보고서까지 연결하는 참고 서비스입니다. 직접 데이터/API 연계는 해당 서비스의 API·이용약관·라이선스 확인 전에는 수행하지 않습니다.

반영 우선순위는 다음과 같습니다.

- 시설 lifecycle과 사업자 유형을 현재 후보지 모델에 추가
- 시설별 MW, 출처, 확인일, confidence, campus/facility 관계를 저장
- 전력·수자원·대수층·환경·지역사회 영향 레이어를 동일 지도에서 비교
- 지도·요약 카드·시뮬레이션 결과가 같은 필터 집합을 공유
- 수치와 위치에 출처·갱신일·불확실성을 항상 표시
