# GRID / SITING LAB 작업 계획

## 목표

Open Infrastructure Map의 대한민국 전력망/발전량 스냅샷을 기반으로 AIDC(Artificial Intelligence Data Center) 후보지를 1차 스크리닝하고, 전력 수요 시나리오를 조절해 후보지의 적합도와 리스크를 비교하는 맵 애플리케이션을 만든다.

## 범위

- 대한민국 전력망: 발전소, 발전량, 154/345/765 kV 전력선, 변전 허브
- AIDC 입지: 계통 접근성·수전 여유·냉각 수자원·토지/인허가·재해/기후
- 배후 인프라: 도시·도로·철도·공항·초고속 광통신 백본·해저케이블/국제 연결·위성통신 회복력
- 디지털 생태계: 클라우드 리전·기존 IDC·피어링, 대학·R&D·AI 인재권
- 시뮬레이션: IT load, PUE, N/N+1/N+2, 연간 소비전력, 필요 수전용량, 후보지 headroom

## 진행 순서

1. [완료] 빈 작업공간 확인 및 정적 웹 프로토타입 구조 결정
2. [완료] OpenInfraMap South Korea 통계/출처 확인
3. [완료] 지도 중심 UI, 후보지 카드, 적합도 스코어, 수요 시뮬레이터 구현
4. [완료] 도시·교통·통신·위성·클라우드/IDC·대학/R&D 레이어를 지도와 점수 모델에 추가
5. [완료] 계획/에이전트/작업 이력 문서화
6. [완료] Overpass API 후보지 반경 동기화와 공공 API 확장 제안 문서화
7. [완료] 기존 AI Data Center Map 기능·데이터 구조 조사 및 연계 우선순위 정리
8. [완료] 반응형 UI, 정적 서버 실행, 브라우저 콘솔 오류 및 주요 상호작용 검증 (Playwright)
9. [완료] 시설 레지스트리(lifecycle/사업자 유형/출처/confidence) 모델과 FACILITIES 레이어 구현
10. [완료] 수자원·환경 레이어와 시뮬레이션 확장 (냉각 방식 WUE, 부하율, 용수, 탄소, 전력비용)
11. [완료] OSM/Overpass 전국 전력망 ETL 스냅샷 (송전선 3,652 · 발전소 4,129) 지도 표출 및 반경 50 km 실측 프록시 시뮬레이션 연동
12. [완료] GitHub 저장소 생성 및 GitHub Pages 배포

## 데이터 원칙

- OpenInfraMap은 OpenStreetMap 기반의 공개 인프라 표현으로 사용한다.
- 국가 통계는 화면에 스냅샷 날짜를 표시한다.
- 위치·용량·접속성의 확정 사실과 탐색용 프록시를 구분한다.
- 후보지 점수는 사업성·계통접속을 확정하지 않는 초기 스크리닝 모델로 고지한다.
- 실제 적용 시 Overpass/OSM 추출, 한전 접속 가능용량, 통신사 경로, 용수·토지·재해·인허가 데이터로 교체할 수 있는 구조를 유지한다.
